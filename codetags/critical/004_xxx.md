# XXX

**Severidade**: 🔴 Critical
**Categoria**: Alerta e Aviso
**Resolver**: Antes do merge

## Definição

Marca código que requer **atenção crítica imediata** - indica algo perigoso, problemático ou que pode causar problemas graves. É um "grito de alerta" no código que não deve ser ignorado.

## Quando Usar

| Situação | Exemplo |
|----------|---------|
| Código extremamente frágil | Pode quebrar com qualquer mudança |
| Lógica perigosa não óbvia | Comportamento contra-intuitivo |
| Dependência de ordem crítica | Sequência que não pode ser alterada |
| Armadilha para desenvolvedores | Código que parece OK mas não é |

## Quando NÃO Usar

| Situação | Use em vez |
|----------|------------|
| Bug confirmado | FIXME ou BUG |
| Vulnerabilidade de segurança | SECURITY |
| Código temporário | HACK |
| Código a ser refatorado | REFACTOR |

## Formato

```javascript
// XXX: ALERTA - descrição do perigo
// XXX: NÃO ALTERAR - razão crítica
// XXX: CUIDADO - explicação do risco
```

## Exemplos

### Exemplo 1: Ordem Crítica de Execução

```javascript
// XXX: ORDEM CRÍTICA - estas linhas DEVEM executar nesta sequência
// Inverter causa race condition que corrompe dados do usuário
await lockAccount(userId);
await processPayment(userId, amount);
await unlockAccount(userId);
// Se unlockAccount executar antes de processPayment, double-charge ocorre
```

### Exemplo 2: Comportamento Contra-Intuitivo

```javascript
// XXX: CUIDADO - esta função MODIFICA o array original
// Parece retornar novo array mas muta in-place por performance
// NÃO passar array que precisa ser preservado
function processItems(items) {
  items.sort((a, b) => a.priority - b.priority);
  items.splice(0, items.length / 2);
  return items; // Mesmo array, modificado!
}
```

### Exemplo 3: Código Frágil

```javascript
// XXX: FRÁGIL - depende de timing específico do browser
// Funciona porque o DOM atualiza em ~16ms
// Qualquer mudança pode quebrar silenciosamente
setTimeout(() => {
  element.classList.add('visible');
}, 20); // NÃO ALTERAR este valor
```

### Exemplo 4: Dependência Externa Crítica

```javascript
// XXX: NÃO ATUALIZAR - versão 2.x quebra compatibilidade silenciosamente
// API externa mudou formato de resposta sem aviso
// Manter exatamente esta versão até migração completa
import { processData } from 'legacy-lib@1.9.3';
```

### Exemplo 5: Efeito Colateral Oculto

```javascript
// XXX: EFEITO COLATERAL - esta query também atualiza cache global
// Parece uma simples leitura mas invalida cache de outros usuários
// Não chamar em loops ou operações frequentes
async function getUserProfile(id) {
  const user = await db.users.findById(id);
  globalCache.invalidate(`user:${id}`); // Efeito colateral!
  return user;
}
```

### Exemplo 6: Armadilha de Configuração

```javascript
// XXX: ARMADILHA - DEBUG=true em produção causa vazamento de memória
// Logs são retidos indefinidamente quando debug está ativo
// NUNCA habilitar em produção, mesmo temporariamente
const DEBUG = process.env.DEBUG === 'true';
if (DEBUG) {
  allLogs.push(logEntry); // Array cresce infinitamente
}
```

## Ação Esperada

1. **Ler** o aviso com atenção antes de modificar
2. **Entender** completamente o risco
3. **Consultar** autor original se possível
4. **Testar** exaustivamente se precisar modificar
5. **Documentar** se o risco foi mitigado
6. **Remover** XXX somente quando o código for reescrito de forma segura

## Resolução

| Contexto | Ação |
|----------|------|
| Código novo com XXX | Reescrever antes do merge |
| Código legado com XXX | Planejar refatoração |
| XXX inevitável | Manter e documentar bem |

## Anti-Patterns

```javascript
// ❌ XXX sem explicação
// XXX:
function dangerous() { }

// ❌ XXX vago
// XXX: cuidado aqui
function dangerous() { }

// ❌ Usar XXX para TODO
// XXX: implementar depois
function incomplete() { } // Use TODO

// ❌ XXX para código comum
// XXX: função de cálculo
function normalFunction() { } // Não precisa de XXX
```

## Diferença de Outros Tags Críticos

| Tag | Foco | Ação |
|-----|------|------|
| FIXME | Bug a corrigir | Consertar o bug |
| BUG | Defeito documentado | Resolver via ticket |
| SECURITY | Vulnerabilidade | Corrigir urgente |
| **XXX** | Código perigoso | Entender antes de tocar |
