# BUG

**Severidade**: 🔴 Critical
**Categoria**: Alerta e Aviso
**Resolver**: Sprint atual

---

## Definição

Documenta um **defeito conhecido** que causa falha ou comportamento inesperado, mas que está sendo rastreado e será corrigido em momento planejado. Diferente de FIXME (correção imediata), BUG indica um problema documentado com ticket/issue associado.

## Quando Usar

| Situação | Exemplo |
|----------|---------|
| Defeito com ticket aberto | Bug reportado e priorizado no backlog |
| Problema conhecido com workaround | Usuários contornam temporariamente |
| Bug de terceiros aguardando fix | Dependência com issue aberta |
| Problema intermitente em investigação | Race condition difícil de reproduzir |

## Quando NÃO Usar

| Situação | Use em vez |
|----------|------------|
| Bug que precisa de fix imediato | FIXME |
| Código funciona mas é ruim | REFACTOR |
| Problema de segurança | SECURITY |
| Código temporário | HACK |

## Formato

```javascript
// BUG: descrição do problema - ticket/issue
// BUG: [JIRA-123] descrição do comportamento incorreto
// BUG: descrição - workaround: como contornar
```

## Exemplos

### Exemplo 1: Bug com Ticket

```javascript
// BUG: [PROJ-456] toast notification não aparece no Safari iOS
// Workaround: usuários podem ver notificações na página de histórico
function showNotification(message) {
  toast.show(message); // Não funciona em Safari iOS < 15
}
```

### Exemplo 2: Bug de Dependência

```javascript
// BUG: date-fns retorna data errada para timezone GMT-3
// Issue: https://github.com/date-fns/date-fns/issues/1234
// Aguardando fix na próxima release
function formatDate(date) {
  return format(date, 'dd/MM/yyyy'); // Pode estar 1 dia errado
}
```

### Exemplo 3: Race Condition

```javascript
// BUG: race condition quando múltiplos requests chegam simultaneamente
// Ocorre em ~2% dos casos de alto tráfego
// Investigação em andamento - ticket PERF-789
async function processOrder(order) {
  const inventory = await getInventory();
  // Inventory pode mudar entre get e update
  await updateInventory(order);
}
```

### Exemplo 4: Bug Conhecido com Workaround

```javascript
// BUG: autocomplete não funciona em campos de senha no Chrome
// Comportamento do browser, não do código
// Workaround: campo hidden com autocomplete="username" antes
function LoginForm() {
  return (
    <>
      <input type="hidden" autoComplete="username" />
      <input type="password" autoComplete="current-password" />
    </>
  );
}
```

## Ação Esperada

1. **Documentar** o bug com descrição clara
2. **Criar ticket** no sistema de rastreamento
3. **Adicionar referência** ao ticket no comentário
4. **Documentar workaround** se existir
5. **Priorizar** correção no backlog
6. **Remover** comentário quando bug for corrigido

## Resolução

| Contexto | Quando Resolver |
|----------|-----------------|
| Bug crítico afetando usuários | Sprint atual |
| Bug com workaround aceitável | Próximas 2 sprints |
| Bug de baixo impacto | Backlog priorizado |
| Bug de terceiros | Monitorar e atualizar |

## Busca no Código

```bash
# Encontrar todos os BUGs
grep -rn "BUG:" src/

# BUGs com tickets
grep -rn "BUG:.*\[" src/

# BUGs sem ticket (precisam de documentação)
grep -rn "BUG:" src/ | grep -v "\["
```

## Anti-Patterns

```javascript
// ❌ BUG sem descrição
// BUG:
function broken() { }

// ❌ BUG sem ticket
// BUG: algo está errado
function broken() { }

// ❌ Usar BUG para code smell
// BUG: código confuso
function works() { } // Use REFACTOR

// ❌ BUG muito antigo sem ação
// BUG: (2019) nunca foi corrigido
function stillBroken() { }
```

## Diferença entre FIXME e BUG

| Aspecto | FIXME | BUG |
|---------|-------|-----|
| Urgência | Imediata | Planejada |
| Ticket | Opcional | Obrigatório |
| Resolver | Antes do merge | Sprint planejada |
| Contexto | Código novo | Código existente |
