---

JavaScript Vanilla não é fácil. JavaScript Vanilla é a linguagem mais **mal compreendida**, mais **subestimada** e mais **poderosa** que existe rodando nativamente em bilhões de dispositivos agora mesmo — enquanto você depende de um framework pra fazer um `toggle` de classe.

Você não usa Vanilla porque é difícil. Usa framework porque é **conveniente**. E conveniência tem um preço: você não sabe o que está acontecendo.

Não sabe por que seu `this` sumiu.
Não sabe por que seu `useEffect` dispara duas vezes.
Não sabe o que o bundle está realmente enviando pro browser.
Não sabe nada.

**E tudo bem — a maioria não sabe.**

---

Este vault é pra quem quer saber.

Vinte anos de JavaScript destilados em notas técnicas sobre o que realmente acontece por baixo: event loop, prototype chain, closures, motor V8, coerção de tipos, Web APIs nativas e tudo que seu framework favorito faz questão de esconder de você.

Abra como vault no **Obsidian**. Explore o grafo. Puxe um fio.

E quando você chegar no fundo, vai entender por que os melhores desenvolvedores frontend do mundo ainda escrevem Vanilla — não porque não conhecem os frameworks, mas porque os **conhecem bem demais**.
